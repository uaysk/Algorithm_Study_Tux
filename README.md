# Algorithm & Data Structure Study

자료구조 및 알고리즘 스터디 과제 공지 및 제출을 위한 레포입니다.

> 수백 개를 무작정 푸는 것보다, **한 문제를 제대로 이해하는 것**이 목표입니다.

> 문제 해결 능력을 향상시키는 것이 목적입니다.

## 🔍 필독

Pull Request를 작성하기 전, 아래 **3단계 사고 프로세스**를 반드시 거쳐야 합니다.

코드를 먼저 작성하기보다는 아래 순서를 따라주세요.

---

### Step 1. 문제 인식 (Perceive)

문제를 풀기 전, 다음 항목을 명확히 정리합니다.

### [아래는 예시일뿐입니다.]
- **Input / Output**: 무엇이 들어오고 무엇이 나오는가?
- **제약 조건**: 입력 범위, 시간/공간 복잡도 요구사항
- **Edge Case**: 빈 배열, 음수, 중복, null 등 예외 상황

---

### **Step 2. 어떤 자료구조나 알고리즘을 사용하는지 결정

이 문제를 해결하기 위해 어떤 자료구조 / 알고리즘을 사용할지 결정

### **Step 3. 의사코드 작성 (Pseudocode First)

풀이 로직을 **코드가 아닌 의사코드(pseudocode)** 로 먼저 작성합니다.

> 코딩 전에 pseudocode로 사고 과정을 보여주는 것이 목적입니다.

> 의사코드는 **Pull Request Description**에 작성합니다.

```
의사코드(Pseudo Code)

- 알고리즘이나 프로그램의 로직을 프로그래밍 언어의 문법이 아닌, 사람의 일상 언어(자연어)와 구조적 프로그래밍의 형식을 섞은 표현방법
- 결국 친숙한 언어나 간략한 코드를 통해 문제 해결 논리를 설명하는 것입니다.
```

---

### Step 4. 풀이 로직 설명 (Verbalize)

스터디에서 모든 인원이 **말로 풀이 로직을 설명**합니다.

- 문제를 어떻게 인식했는가?
- 왜 이 자료구조/알고리즘을 선택했는가?
- 어떤 흐름/로직을 통해 해결했는지
- 시간 복잡도는 어떻게 계산했는가?

---

## 📥 과제 제출 방법

```
1. 레포를 fork 후 git clone으로 로컬에 내려받기
2. git checkout -b {본인이름} 으로 브랜치 생성
3. 기능 단위로 git commit 및 git push
4. 완성 후 Pull Request 생성
```
